## 3. Credentials propres dès le premier jour

### 3.1 Pourquoi pas un token « navigateur »

Sans OAuth dans la CLI, le token « rapide » s'obtient via Graph API Explorer
(developers.facebook.com/tools/explorer) : token **utilisateur** de 1-2 h,
prolongeable à ~60 jours (bouton « Extend Access Token » dans l'Access Token
Debugger). Lié à ton login personnel, il expire et casse silencieusement les
sessions moteur. **On passe directement au token System User : n'expire
jamais, appartient au portfolio (pas à une personne), périmètre minimal.**
C'est LE token que toute session moteur future utilisera
(`META_SYSTEM_USER_TOKEN`), jamais une session navigateur.

### 3.2 Préalable unique : une app Meta (~5 min, clic par clic)

Le « Generate token » d'un system user exige une app liée au portfolio.

1. Ouvre **https://developers.facebook.com** → bouton **« Log in »** en haut à
   droite → connecte-toi avec le compte Facebook **admin du portfolio TAQINOR**.
2. Première visite : bouton **« Get Started »** → confirme l'email → rôle
   **« Developer »**.
3. En haut à droite : **« My Apps »** → bouton vert **« Create App »**.
4. Écran « What do you want your app to do? » (ou « Add use cases ») : choisis
   **« Other »** → **Next** → « Select an app type » : **« Business »** →
   **Next**. (Si le wizard propose directement un use case « Business », c'est
   celui-là.)
5. « Add details » : App name **`TAQINOR Engine`**, App contact email : ton
   email.
6. **« Business portfolio »** (menu déroulant) : **sélectionne le portfolio
   TAQINOR** — étape critique : sans ce lien, l'app n'apparaîtra pas dans le
   dialogue de génération du token (§3.3 étape 6).
7. **« Create app »** → Meta redemande ton mot de passe → l'app existe.
8. Rien d'autre : aucun « product » à ajouter, l'app reste en **Development
   mode** (suffisant pour lire les assets de TON propre business).

### 3.3 Token System User « never expires » (clic par clic)

1. **https://business.facebook.com** → vérifie en haut à gauche que le
   portfolio TAQINOR est sélectionné.
2. Engrenage **« Settings »** en bas à gauche (ou directement
   **https://business.facebook.com/settings**). Si tu atterris dans le
   « Meta Business Suite → Settings » nouveau style, clique le lien
   **« Business settings »** (parfois « More business settings ») pour ouvrir
   l'écran classique.
3. Menu de gauche : **Users → System users** (UI récente : « System
   accounts ») → bouton bleu **« Add »**. (Première fois : accepter les
   conditions plateforme.)
4. Dialogue : System username **`taqinor-engine`** ; rôle :
   **« Employee »** (PAS Admin — minimal) → **« Create system user »**.
5. `taqinor-engine` sélectionné → bouton **« Add assets »** (ou « Assign
   assets ») — trois passages :
   - **Ad accounts** → coche ton compte pub → à droite, active
     **« Manage campaigns »** (niveau lecture + gestion demandé pour le moteur ;
     « View performance » serait la lecture stricte). Laisse « Manage ad
     account » (full control) **OFF** → **Save**.
   - **Pages** → ta Page Facebook → active **« Create ads »** et
     **« View insights »** (accès partiel, full control OFF) → **Save**.
     La Page DOIT être assignée, sinon pas de lecture des Instant Forms.
   - **Datasets** (parfois listé « Pixels ») → ton pixel → **« View »** →
     **Save**.
6. Retour sur la fiche du system user → **« Generate new token »** :
   - **Select app** : `TAQINOR Engine` (si la liste est vide → l'app n'est pas
     liée au portfolio, retour §3.2 étape 6).
   - **Token expiration : « Never »** ← tout l'intérêt.
   - Coche les permissions (jeu minimal lecture + gestion ads + leads) :
     `ads_read`, `ads_management`, `business_management`, `pages_show_list`,
     `pages_read_engagement`, `pages_manage_ads`, `leads_retrieval`,
     `read_insights`.
     (`ads_management` = le volet « manage » demandé ; pour un token
     strictement lecture, ne pas le cocher. `pages_manage_ads` +
     `leads_retrieval` permettent au script de récupérer le token de Page et
     de télécharger les leads des Instant Forms.)
   - **« Generate token »** → **copie-le IMMÉDIATEMENT** (affiché une seule
     fois, commence par `EAA...`).
7. Révocation/rotation possible à tout moment depuis le même écran
   (« Revoke » puis regénérer, puis re-stocker via §3.4).

**Gotcha leads** : si plus tard SEUL l'export de leads échoue en permission
(#200), ouvre Meta Business Suite → All tools → **Instant Forms → Leads access**
(Leads Access Manager) : si « Customize access » a été activé un jour sur la
Page, ajoute le system user / l'app. Par défaut ce n'est pas nécessaire.

### 3.4 Stockage dans le `.env` du serveur ERP (depuis TON PC, même accès que
`scripts/deploy-prod.ps1`)

La clé `~/.ssh/taqinor_hetzner` n'existe QUE sur le PC du fondateur (vérifié :
absente de la session cloud — c'est un blocage credential assumé, pas un oubli).
Depuis PowerShell sur le PC :

```powershell
$T = Read-Host "Coller META_SYSTEM_USER_TOKEN"
$T | ssh -i "$env:USERPROFILE\.ssh\taqinor_hetzner" root@178.105.192.116 'read -r T; touch /opt/taqinor-os/.env; chmod 600 /opt/taqinor-os/.env; if grep -q "^META_SYSTEM_USER_TOKEN=" /opt/taqinor-os/.env; then sed -i "s|^META_SYSTEM_USER_TOKEN=.*|META_SYSTEM_USER_TOKEN=$T|" /opt/taqinor-os/.env; else printf "META_SYSTEM_USER_TOKEN=%s\n" "$T" >> /opt/taqinor-os/.env; fi; grep -c "^META_SYSTEM_USER_TOKEN=" /opt/taqinor-os/.env'
```

Sortie `1` = stocké. Le token ne passe ni dans l'historique PowerShell
(Read-Host) ni dans l'historique shell du serveur (stdin). Aucun redémarrage
de conteneur nécessaire aujourd'hui : compose relira `.env` au prochain
`up -d`, et le script d'audit lit le fichier directement. Le token ne va
JAMAIS dans le dépôt ni dans un `.env` versionné. (Ajout d'une ligne
`META_SYSTEM_USER_TOKEN=` à `.env.example` : hors périmètre docs-only de cette
session — à faire au prochain commit backend.)

## 9. Verdict : GARDER la campagne, RECÂBLER le circuit

En langage simple, chiffres §5-8 à l'appui :

**La machine à leads marche et n'est pas chère — on ne coupe pas, on ne
remplace pas.** 3 $/jour donnent 59 leads en 30 jours à **1,53 $ (≈ 15 MAD)
le lead**, ce sont de vrais leads (objectif OUTCOME_LEADS, pas des clics),
le formulaire pose de vraies questions qualifiantes (facture mensuelle,
ville, délai du projet), l'audience est saine (fréquence 2,2 < 3, Casablanca
+ 40 km) et le flux est régulier (~2/jour, le dernier lead date du jour de
l'audit). Pour du solaire résidentiel où un devis signé se compte en dizaines
de milliers de MAD, ce coût par lead est excellent — tous les critères
« GARDER » de la grille d'origine sont verts.

**Mais tout ce qui se passe APRÈS le formulaire est cassé — d'où RECÂBLER**
(la grille disait : coût correct MAIS synchro CRM absente / pixel muet /
créa unique — les trois sont constatés) :

1. **Les leads pourrissaient dans Meta.** 88 des 240 leads avaient dépassé
   60 jours, le plus ancien à 1 jour de l'effacement définitif (Meta purge à
   90 j). L'export CSV du 2026-07-24 est un sauvetage, pas une solution :
   il faut faire atterrir chaque nouveau lead dans le CRM de l'ERP (webhook
   `leadgen` ou relève programmée — le CRM et le circuit webhook côté site
   existent déjà ; c'est le chantier ENG/ADSDEEP déjà au plan).
2. **Le pixel n'a jamais servi** (§8) : zéro événement depuis sa création.
   Meta ne sait pas ce qu'un lead devient (devis envoyé ? signé ?) et ne peut
   pas optimiser au-delà de la soumission du formulaire. L'installer sur
   taqinor.ma, puis brancher la CAPI avec les statuts de l'ERP.
3. **Une seule créa depuis 3 mois** : 4 annonces actives = la même accroche
   copiée 4 fois, 1 marquée WITH_ISSUES. La fréquence (2,2) est encore saine
   mais montera mécaniquement ; préparer 2-3 variantes AVANT l'usure.
4. (Mineur) Compte facturé en **USD** avec un budget minuscule (3 $/j, pas de
   plafond de dépense) : le jour où le circuit CRM est branché et que la
   valeur d'un lead est mesurée dans l'ERP, c'est LE levier à monter — pas
   avant.

Rappel de la grille appliquée : **GARDER** = coût/lead nettement sous la
valeur d'un lead solaire + fréquence < 3 + flux régulier + formulaire
qualifiant (tous constatés) ; **RECÂBLER** = leads qui pourrissent dans Meta,
pixel muet, créa unique (tous constatés aussi) ; **REMPLACER** = coût
déraisonnable ou résultats = clics (aucun des deux). D'où : garder la
campagne telle quelle, recâbler ciblage aval / tracking / créa autour d'elle.
